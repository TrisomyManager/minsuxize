## 🗂️ 本地版本控制策略

### Git 分支策略
```
main (主分支，稳定版本)
├── develop (开发分支)
│   ├── feature/data-model-extensions (数据模型扩展)
│   ├── feature/submission-system (投稿系统)
│   ├── feature/review-workflow (审核工作流)
│   └── feature/version-control (版本控制)
└── hotfix/* (紧急修复)
```

### 提交规范
```
feat: 新功能
fix: 修复bug
docs: 文档更新
style: 代码格式调整
refactor: 代码重构
test: 测试相关
chore: 构建过程或辅助工具的变动
```

### 本地开发流程
1. 从 `develop` 分支创建特性分支
2. 在特性分支上开发
3. 完成功能后合并到 `develop`
4. 定期从 `main` 合并到 `develop`
5. 发布版本时从 `develop` 合并到 `main`

## 🔧 开发环境配置

### 必需工具
- .NET SDK 8.0+
- Visual Studio Code 或 Visual Studio
- SQLite 数据库工具
- Git

### 可选工具
- Docker (用于容器化部署)
- Postman (API 测试)
- Swagger UI (API 文档)
- Leaflet/Mapbox (地图集成)

### 环境变量配置
创建 `.env` 文件（不提交到版本控制）：
```bash
# 数据库配置
DATABASE_CONNECTION=Data Source=./App_Data/minsuxize.db

# 管理员配置
ADMIN_USERNAME=admin
ADMIN_PASSWORD=your_secure_password_here

# 文件上传配置
MAX_FILE_SIZE=10485760  # 10MB
ALLOWED_EXTENSIONS=.jpg,.jpeg,.png,.gif,.mp4,.mp3,.pdf,.doc,.docx
```
