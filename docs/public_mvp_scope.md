# 公众 MVP 范围说明

## 公众 MVP 目标

在现有 ASP.NET Core MVC 站点基础上，尽快形成一个能上线试运行的公众版最小产品，让用户可以：

- 浏览公开的地方习俗实例
- 通过列表和详情页理解内容结构
- 通过反馈页补充、纠错或提交新线索
- 让团队在轻后台里看到真实投稿与最小审核状态

## 首期做什么

- 公众首页
- 习俗列表页
- 习俗详情页
- 反馈/补充提交页
- 轻后台登录与提交队列
- SQLite 样板数据接入与持续复用

## 首期不做什么

- 复杂地图系统
- 图谱可视化
- 扩展实体独立建模
- 复杂审核后台
- 复杂版本历史能力
- 正式媒体上传系统
- 多角色权限体系

## 为什么这样收缩

- 现有仓库已经有 MVC 壳子、SQLite、样板数据、前后台页面雏形，直接收缩比重做更快。
- 当前最缺的不是更多概念建模，而是一个能被公众访问、能收到反馈、能验证内容结构的运行闭环。
- 先把“地方习俗实例”跑通，才能知道后续到底要不要扩展地图、图谱、复杂知识关系。

## 首期验收标准

- 首页能清楚表达站点定位，并提供浏览与提交入口
- 列表页能按关键词、地区、节日筛选公开习俗实例
- 详情页能展示实例内容、来源和补充入口
- 反馈页能提交到数据库，并进入后台队列
- 轻后台能登录、查看提交、变更最小状态
- 项目可以稳定构建并本地启动
