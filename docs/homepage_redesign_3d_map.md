# 民俗网站首页重设计方案
## 三维中国地图 + 民俗标签系统

---

## 1. 核心概念

### 视觉理念
- **宏观视角**：用户首次进入看到完整的中国三维地形
- **探索发现**：通过旋转、缩放、 hover 发现各地民俗
- **层次递进**：国家 → 省份 → 城市 → 具体民俗条目

### 交互逻辑
```
用户路径：
进入首页 → 看到三维中国地图 → 鼠标悬停省份高亮 → 
显示该地特色民俗标签 → 点击标签进入详情/点击省份放大 → 
展示该省更多民俗点
```

---

## 2. 技术方案选型

### 方案A：Three.js + GeoJSON（推荐）
| 优点 | 缺点 |
|------|------|
| 完全可控的3D效果 | 开发复杂度较高 |
| 可定制地形高度 | 需要处理性能优化 |
| 支持复杂交互 | 地图数据需要处理 |
| 无第三方依赖 | 学习曲线较陡 |

**技术栈**：
- Three.js - 3D渲染
- D3.js - GeoJSON数据处理和投影
- TopoJSON - 地图数据压缩
- 阿里云DataV或高德地图边界数据

### 方案B：ECharts GL
| 优点 | 缺点 |
|------|------|
| 开箱即用 | 定制性受限 |
| 文档完善 | 3D效果较基础 |
| 性能较好 | 大地图数据量大会卡顿 |
| 中文支持好 | 风格较传统 |

### 方案C：Mapbox GL JS + 自定义图层
| 优点 | 缺点 |
|------|------|
| 专业地图引擎 | 需要API Key |
| 底图精美 | 定制费用 |
| 3D地形支持 | 数据隐私 |

## 3. 推荐方案详细设计（Three.js）

### 3.1 地图呈现

#### 地形效果
```
- 使用真实地形高度数据（SRTM或简化版）
- 省份边界发光描边
- 海洋使用半透明蓝色平面
- 整体色调：国风暗色系（深蓝、暗金、朱红点缀）
```

#### 视觉风格参考
- **主色调**：墨黑背景 `#0a0a0f`
- **地形色**：深蓝渐变 `#1a1a2e` → `#16213e`
- **高亮色**：朱红/金色 `#c41e3a` / `#ffd700`
- **标签色**：半透明深色背景 + 白色文字

### 3.2 民俗标签系统

#### 标签类型层级
```
一级标签（省级）：
- 显示该省最知名的1-2个民俗
- 大图标 + 文字
- 位置：省份中心点

二级标签（市级）：
- 缩放后出现
- 中等图标
- 位置：城市中心

三级标签（具体民俗）：
- 再次缩放/点击城市后出现
- 小圆点 + tooltip
- 位置：具体地点坐标
```

#### 标签内容
```typescript
interface FolkTag {
  id: string;           // 民俗条目ID
  name: string;         // 民俗名称（如"春节贴春联"）
  emoji: string;        // 图标（如🧧）
  region: string;       // 所属地区
  category: string;     // 分类（节日/婚俗/饮食/服饰...）
  hotLevel: 1-5;        // 热门程度（影响标签大小）
  hasDetail: boolean;   // 是否有详情页
  coordinates: [x, y, z]; // 3D坐标
}
```

#### 标签视觉设计
```
热门民俗（5星）：
┌──────────────────┐
│ 🧧 春节贴春联     │  ← 金色边框，微微发光
│    点击查看 →     │  ← 呼吸灯效果
└──────────────────┘

普通民俗（3星）：
┌────────────┐
│ 🏮 赏花灯   │  ← 白色边框
└────────────┘

小众民俗（1星）：
● 点雄黄      ← 仅小圆点，hover显示详情
```

## 4. 动画效果

### 入场动画
```
1. 黑色背景淡入（0.5s）
2. 中国轮廓从中心放大出现（1s, ease-out）
3. 省份边界依次点亮（0.1s/省，从沿海向内陆）
4. 热门民俗标签从省份中心弹出（0.3s/个，错峰）
5. 整体旋转展示3D效果（持续缓慢自转）
```

### 交互动画
```
Hover省份：
- 省份表面亮度+20%
- 边界发光增强
- 省份名称卡片从边界滑入

点击省份：
- 相机平滑移动（1s，ease-in-out）
- 该省其他民俗标签依次出现
- 周边省份渐暗

标签Hover：
- 缩放1.1倍
- 向上浮动10px
- 边框发光

标签点击：
- 标签闪烁
- 详情面板从右侧滑入
- 相机轻微调整聚焦
```

## 5. 性能优化

### 地图数据优化
```
- 使用TopoJSON压缩地理数据
- 省份mesh合并减少draw call
- 简化海岸线精度（开发时可用低精度版）
- 动态加载：根据相机距离加载不同精度
```

### 标签优化
```
- LOD系统：远距离只显示热门标签
- 标签碰撞检测：避免重叠
- 对象池复用标签DOM/Sprite
- 节流：缩放时防抖更新标签
```

### 渲染优化
```
- 使用InstancedMesh批量渲染相同元素
- 开启Frustum Culling
- 限制最大标签数量（50个）
- 离屏标签禁用交互
```

## 6. 响应式设计

### 桌面端（>1024px）
- 完整3D地图
- 右侧详情面板
- 底部筛选栏
- 支持精细操作

### 平板端（768-1024px）
- 3D地图简化（减少标签数量）
- 底部详情面板（半屏弹出）
- 筛选按钮收起为图标

### 移动端（<768px）
- 提供2D简化版选项（性能考虑）
- 3D版：限制旋转范围，主要自动播放
- 点击省份后全屏展示该省民俗
- 详情页面跳转（非面板）

## 7. 数据接口

### 需要的API
```typescript
// 获取省份列表及中心坐标
GET /api/regions?level=province
→ [{ id, name, center: [lat, lng], bounds }]

// 获取指定区域的民俗条目
GET /api/folklore?region={regionId}&category={cat}&limit=10
→ [{ id, name, emoji, coordinates, hotLevel }]

// 搜索民俗
GET /api/folklore/search?q={keyword}
→ [{ id, name, region, coordinates }]

// 获取民俗详情
GET /api/folklore/{id}
→ 完整条目数据
```

## 9. 技术风险与应对

| 风险 | 可能性 | 应对方案 |
|------|--------|----------|
| 地图数据文件过大 | 中 | 使用TopoJSON压缩，分片加载 |
| 低配置设备卡顿 | 高 | 提供2D简化版，自动降级 |
| 标签重叠严重 | 中 | 实现标签聚类算法 |
| 学习成本高 | 中 | 先用ECharts原型验证 |
| 移动端性能差 | 高 | 移动端切换为列表视图 |

## 10. 备选简化方案

如果3D实现过于复杂，可先采用：

### 2.5D地图方案
- 使用ECharts GL的3D地图
- 省份块柱状高度表示民俗数量
- 顶部显示标签

### SVG交互地图
- D3.js绘制可交互中国地图
- 省份hover效果
- 点击展开详情
- 实现简单，性能好
