# 项目主说明书

## 1. 项目名称
中国传统文化习俗数字记录与知识系统

## 2. 项目定位
本项目旨在建立一套以“地方习俗实例”为核心的知识工程，用于持续记录中国传统文化习俗，并支持未来转化为：
- 维基式知识网站
- 地图展示平台
- 节令时间线平台
- 知识图谱系统
- 数字地方志与专题策展系统

## 3. 核心原则
1. 以地方实例为核心，而不是只记录抽象节日名词。
2. 抽象词条与地方实例分离。
3. 所有核心记录绑定地点、时间与来源。
4. 允许多版本并存，不强行统一地方说法。
5. 先做试点，再扩展全国。

## 4. 记录对象层级
### 4.1 抽象知识层
- 节日
- 仪式
- 器物
- 人群角色
- 信仰观念
- 时间规则

### 4.2 地方实例层
- 某地某节日的具体做法
- 某地某仪式的流程变体
- 某地某人生礼仪的地方实践

### 4.3 证据资料层
- 地方志
- 论文
- 采访
- 口述史
- 照片
- 视频
- 音频
- 扫描件

## 5. 首期范围
### 5.1 首期建议实体
- 地点
- 节日
- 仪式
- 地方习俗实例
- 来源资料

### 5.2 首期建议功能映射
- 词条页
- 地图入口
- 节令入口
- 基础关系图

### 5.3 首期建议规模
- 1 个试点区域
- 10–30 个村级地点
- 30–100 条地方习俗实例
- 50–200 条来源资料

## 6. 明确不做的内容
- 不在首期做全国全量覆盖
- 不在首期展开数据库实现细节
- 不在没有来源时编造地方习俗事实
- 不混写抽象节日词条和地方实例词条

## 7. 阶段划分
- Phase 0：项目底座
- Phase 1：知识结构
- Phase 2：内容模板
- Phase 3：样板工程
- Phase 4：产品映射
- Phase 5：协作运营
