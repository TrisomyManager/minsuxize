# 模块队列

## 文档信息
- 文档路径：`00_Project_Brief/module_queue.md`
- 对应阶段：`Phase 0` 至 `Phase 5`
- 当前版本：`v1.5.0`
- 当前状态：`approved`

## 1. 使用说明
- 本文档是项目推进清单，不是概念说明文档。
- 每次会话只能领取一个模块，除非模块本身明确标记为需拆分执行。
- 领取前必须确认输入依赖已达到 `approved` 或至少不阻塞本轮目标。
- 模块完成后，必须同步更新 `00_Project_Brief/current_status.md`、`05_Operations/change_log.md` 和目标文件内的修改记录。
- 如发现上游冲突，先登记 `05_Operations/review_log.md`，不得直接覆盖。

## 2. 模块状态定义
| 状态 | 含义 | 执行动作 |
|---|---|---|
| `not_started` | 尚未开始正式编写 | 可领取执行 |
| `in_progress` | 已被占用，正在处理 | 其他人不得并行改写同一输出文件 |
| `in_review` | 已有首版，待审核 | 审核型 AI 或人工审核优先处理 |
| `approved` | 已通过审核，可作为下游依赖 | 下游模块可引用 |
| `blocked` | 存在依赖缺失或冲突 | 先处理阻塞项 |
