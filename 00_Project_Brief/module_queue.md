# 模块队列

## 文档信息
- 文档路径：`00_Project_Brief/module_queue.md`
- 对应阶段：`Phase 0` 至 `Phase 5`
- 当前版本：`v1.0.0`
- 当前状态：`in_review`

## 1. 使用说明
- 本文档是项目推进清单，不是概念说明文档。
- 每次会话只能领取一个模块，除非该模块明确标注为需要拆分执行。
- 开始执行前，必须确认输入依赖已具备，且目标文件未被其他人或其他 AI 占用。
- 模块完成后，必须同步更新 `00_Project_Brief/current_status.md`、`05_Operations/change_log.md` 和对应目标文件中的修改记录。
- 若模块输出与上游规范冲突，优先暂停提交并登记到 `05_Operations/review_log.md`，不得跳过冲突直接覆盖。

## 2. 模块状态定义
| 状态 | 含义 | 执行动作 |
|---|---|---|
| `not_started` | 尚未开始正式编写 | 可领取执行 |
| `in_progress` | 已被占用，正在处理 | 其他人不得并行改写同一输出文件 |
| `in_review` | 已有首版，待审核 | 审核型 AI 或人工审核优先处理 |
| `approved` | 已通过审核，可作为下游依赖 | 下游模块可引用 |
| `blocked` | 存在依赖缺失或冲突 | 先处理阻塞项 |

## 5. 模块领取规则
- 领取模块前，必须确认其输入依赖至少达到 `in_review` 状态。
- 若上游模块仍为 `not_started` 或 `blocked`，不得硬推进下游模块。
- 一个会话中若只能完成部分内容，必须在交接中明确“已完成 / 未完成 / 阻塞原因”，不得写成“已完成”。
- 若模块标注为“不适合单次会话完成”，执行前应先切分子任务，但不得改动模块编号。
