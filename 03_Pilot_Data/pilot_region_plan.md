# 试点区域选择框架

## 1. 目标
为首期工程选择一个适合跑通架构和流程的试点区域。

## 2. 评估维度
| 维度 | 说明 | 建议权重 | 评分备注 |
|---|---|---:|---|
| 资料可得性 | 地方志、论文、采访、影像是否较易获取 | 20 | |
| 民俗密度 | 区域内可覆盖的节日、仪式、习俗是否丰富 | 20 | |
| 差异性 | 是否有明显地方特色，便于体现“实例价值” | 20 | |
| 采集可行性 | 交通、联系、实地走访难度 | 15 | |
| 展示价值 | 地图、时间线、图谱展示效果是否明显 | 15 | |
| 持续扩展性 | 后续扩点是否顺畅 | 10 | |

## 3. 首期建议规模
- 村级地点：10–30
- 地方习俗实例：30–100
- 来源资料：50–200

## 4. 风险
- 选区过大导致首期无法闭环
- 资料虽多但质量分散
- 真实采集周期长于预期

## 5. 完成判定
- 已形成首批样板词条
- 已能映射到地图 / 节令 / 图谱入口
- 已验证模板与字段足够支撑扩展
