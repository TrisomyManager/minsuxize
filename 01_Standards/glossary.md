## 1. 地点相关术语

### 1.1 地点（Place）
- **定义**：具有明确地理边界的区域单元，是本项目所有核心记录的空间锚点对象。地点按行政区划层级从高到低依次为：国家 → 省 → 市 → 县/区 → 乡镇/街道 → 行政村 → 自然村 → 空间点位。
- **使用边界**：地点记录必须包含标准化层级名称，优先精确到自然村级别，至少到行政村。地点不负责记录该区域内发生的习俗内容，仅作为空间参照框架。
- **易混概念**：与"地方习俗实例"不同——地点是空间容器，地方习俗实例是在该地点发生的具体习俗实践；与"行政区划"不同——地点允许记录自然村等非正式行政单元。
- **项目内示例**：`福建省莆田市涵江区江口镇`（乡镇级地点）、`福建省莆田市涵江区江口镇某村`（行政村地点）。

### 1.2 行政村（Administrative Village）
- **定义**：经民政部门正式设立的村级行政单位，具有唯一行政区划代码，是村民自治组织的管辖范围。
- **使用边界**：行政村作为地点层级中的一个层级使用，不单独作为实体类型。记录时必须使用官方标准名称，避免使用俗称或历史名称。
- **易混概念**：与"自然村"不同——行政村是行政概念，可能包含多个自然村；自然村是自然形成的居民聚居点，可能跨越行政村边界。
- **项目内示例**：`福建省莆田市涵江区江口镇顶坡村`（行政村）。

### 1.3 自然村（Natural Village）
- **定义**：历史上自然形成、居民聚居而成的村落，是民间习俗实践的实际发生单元，通常比行政村更小、更具体。
- **使用边界**：自然村作为地点层级的最细粒度之一使用。当习俗实践在自然村级别存在差异时，必须精确到自然村。自然村名称可能与行政村重复或嵌套。
- **易混概念**：与"行政村"不同——自然村是地缘概念，不一定有正式行政边界；与"空间点位"不同——自然村仍是一个区域，而非具体坐标点。
- **项目内示例**：`福建省莆田市涵江区江口镇顶坡村后厝自然村`。

### 1.4 空间点位（Spatial Point）
- **定义**：具有精确地理坐标（经纬度）的特定位置，用于标注仪式举行地点、庙宇位置、聚集场所等需要精确空间定位的对象。
- **使用边界**：空间点位作为地点层级的补充，仅在需要精确定位时使用（如某座庙宇、某个祠堂、某棵神树）。不用于泛指整个村落或区域。
- **易混概念**：与"地点"不同——地点是一个区域范围，空间点位是一个坐标点；与"自然村"不同——自然村是居民聚居区域，空间点位是具体位置标记。
- **项目内示例**：`福建省莆田市涵江区江口镇顶坡村后厝自然村某宗祠（经纬度：119.12, 25.46）`。

## 2. 习俗内容相关术语

### 2.1 节日（Festival）
- **定义**：在特定历法时间周期性举行、具有文化意义的日子，以及围绕该日子形成的一整套习俗活动集合。节日是抽象概念，描述"这是什么日子、通常做什么"。
- **使用边界**：节日词条记录通用的、跨地域的节日概念，不记录某地具体怎么做。节日词条用于知识组织和导航，必须与记录具体实践的地方习俗实例分离。
- **易混概念**：与"地方习俗实例"不同——节日是抽象词条（如"春节"），地方习俗实例是具体实践（如"福建省莆田市涵江区正月初四过大年"）。
- **项目内示例**：`春节`——农历新年，传统习俗包括贴春联、吃年夜饭、拜年、放鞭炮等。

### 2.2 仪式（Ritual）
- **定义**：为达成特定目的（祭祀、祈福、过渡、凝聚群体等）而进行的、具有固定程序和行为规范的一套活动流程。仪式是抽象概念，描述"通常怎么做"。
- **使用边界**：仪式词条记录通用的仪式类型和流程框架，不记录某地具体怎么执行。仪式必须与地方习俗实例分离，后者记录具体地点的具体执行变体。
- **易混概念**：与"地方习俗实例"不同——仪式是抽象流程（如"婚礼仪式"），地方习俗实例是具体实践（如"福建省莆田市涵江区某村婚礼流程"）。
- **项目内示例**：`祭祖仪式`——向祖先表达敬意的仪式流程，通常包括摆放供品、上香、跪拜、焚烧纸钱等环节。

### 2.3 地方习俗实例（Custom Instance）
- **定义**：在特定地点、特定时间范围内，某一习俗主题（节日、仪式或其他习俗活动）的具体实践形态。地方习俗实例是本项目最核心的记录单位。
- **使用边界**：每条地方习俗实例必须绑定具体地点、明确时间范围、关联来源资料。不同地点的同一习俗主题产生不同实例，允许多版本并存。不得与抽象词条混写。
- **易混概念**：与"节日/仪式"不同——后者是抽象词条，前者是具体实践；与"地点"不同——地点是空间容器，地方习俗实例是在该地点发生的行为记录。
- **项目内示例**：`福建省莆田市涵江区正月初四过大年`——描述该地点在春节期间的具体过法、流程、参与者、禁忌等。

### 2.4 抽象词条（Abstract Entry）
- **定义**：对习俗概念、分类、流程的概括性描述，不绑定具体地点，用于知识组织和导航。包括节日、仪式、器物类型、信仰观念等。
- **使用边界**：抽象词条负责回答"这是什么"，不回答"某地怎么做"。抽象词条必须与地方习俗实例严格分离，二者不得混写在同一记录中。
- **易混概念**：与"地方习俗实例"相对——前者是通用概念，后者是具体实践；与"实体"不同——抽象词条是实体的一种内容形态，不是实体类型本身。
- **项目内示例**：`拜年`——春节期间晚辈向长辈表达祝福的礼仪行为，通用流程包括作揖、说吉祥话等。

## 3. 扩展实体相关术语（首期不展开）

### 3.1 人群角色（Person Role）
- **定义**：在习俗活动中承担特定功能、身份或责任的人群分类，如主祭人、媒人、长辈、孝子、新娘等。
- **使用边界**：人群角色在首期作为地方习俗实例的文本描述内容，不作为独立实体展开。二期可扩展为独立实体，用于分析不同角色的职责和互动。
- **易混概念**：与"具体人物"不同——人群角色是类型概念（如"主祭人"），不记录真实姓名；与"参与者"不同——参与者是泛指，人群角色有特定功能定义。
- **项目内示例**：`主祭人`——祭祀仪式中负责主持流程、诵读祭文的核心角色，通常由家族长辈或特定身份者担任。

### 3.2 器物（Object）
- **定义**：在习俗活动中使用的物品、工具、服饰、供品等物质载体，具有特定的象征意义和实用功能。
- **使用边界**：器物在首期作为地方习俗实例的文本描述内容，不作为独立实体展开。二期可扩展为独立实体，用于建立器物与仪式、地点的关联。
- **易混概念**：与"普通物品"不同——器物必须在习俗语境中具有文化意义；与"供品"不同——供品是器物的子集，特指祭祀时奉献的物品。
- **项目内示例**：`香炉`——祭祀仪式中用于插香、象征与神灵/祖先沟通的器物。

### 3.3 信仰观念（Belief）
- **定义**：支撑习俗实践的宗教信仰、民间信仰、宇宙观、价值观、禁忌观念等非物质文化内容。
- **使用边界**：信仰观念在首期作为地方习俗实例的文本描述内容，不作为独立实体展开。二期可扩展为独立实体，用于分析不同习俗背后的观念基础。
- **易混概念**：与"仪式"不同——仪式是外在行为，信仰观念是内在认知；与"禁忌"不同——禁忌是信仰观念指导下的行为约束。
- **项目内示例**：`祖先崇拜`——相信祖先灵魂存在并能影响子孙后代的观念，是祭祖习俗的观念基础。

### 3.4 时间规则（Time Rule）
- **定义**：决定习俗活动何时举行的历法规则、周期规则、条件规则等，如节气计算、黄历择日、生命周期节点等。
- **使用边界**：时间规则在首期作为地方习俗实例的文本描述内容，不作为独立实体展开。二期可扩展为独立实体，用于分析时间规则的历法依据和地域差异。
- **易混概念**：与"日期"不同——日期是具体时间，时间规则是确定日期的规则；与"节气"不同——节气是时间规则的一种具体类型。
- **项目内示例**：`黄历择日`——根据农历、干支、吉凶宜忌等规则选择适宜举行仪式的日期。

## 4. 证据与管理相关术语

### 4.1 来源资料（Source）
- **定义**：支撑地方习俗实例事实性的证据材料，包括文献、访谈、影像、口述等。来源资料是确保记录可信度、可核查性的基础。
- **使用边界**：每条关键事实必须绑定来源资料。来源资料作为独立实体管理，可被多条地方习俗实例引用。来源需记录类型、作者、时间、可信度。
- **易混概念**：与"媒体资料"不同——来源资料强调证据属性（用于支撑事实），媒体资料强调内容载体属性（用于展示）；与"参考文献"不同——来源资料包括田野调查等非文献材料。
- **项目内示例**：`《莆田市涵江区志》（2010年版）第245-250页关于春节习俗的记载`。

### 4.2 媒体资料（Media）
- **定义**：以图像、音频、视频、扫描件等形式记录习俗活动的数字或模拟媒体文件，用于展示和辅助理解。
- **使用边界**：媒体资料在首期作为来源资料的子类型或补充，不作为独立实体全面展开。媒体需记录类型、关联对象、版权状态。二期可扩展为独立实体。
- **易混概念**：与"来源资料"不同——来源资料是证据概念，媒体资料是载体概念，二者有交叉但不等同；与"图片库"不同——媒体资料必须与具体地方习俗实例关联，不是独立图片集合。
- **项目内示例**：`某村春节祭祖仪式现场照片（2024年2月拍摄，关联 CUS-0001）`。

## 5. 知识系统相关术语

### 5.1 实体（Entity）
- **定义**：知识系统中具有独立身份、可被唯一标识和引用的对象类型。每个实体类型有定义的属性集合。
- **使用边界**：首期核心实体包括地点、节日、仪式、地方习俗实例、来源资料。二期扩展实体包括人群角色、器物、信仰观念、时间规则、媒体资料。
- **易混概念**：与"对象"不同——实体强调结构化、可标识、可关联；与"词条"不同——词条是展示单位，实体是数据单位，一个词条可能涉及多个实体。
- **项目内示例**：`地点（Place）`——具有层级结构、可精确到自然村的地理实体类型。

### 5.2 字段（Field）
- **定义**：描述实体某一属性的数据单元，具有名称、类型、约束等元数据。字段是结构化录入和检索的基础。
- **使用边界**：每个实体的字段由核心字段表定义。字段名必须使用英文小写加下划线，不得擅自新增未定义字段。必填字段必须填写，选填字段可为空。
- **易混概念**：与"属性"不同——属性是概念，字段是具体实现；与"列"不同——列是数据表概念，字段是 Schema 概念。
- **项目内示例**：`custom_instance_id`——地方习俗实例的唯一标识字段，格式为 `CUS-XXXX`。

### 5.3 关系（Relation）
- **定义**：两个实体之间的语义连接，描述它们之间的逻辑关联。关系有方向性（主语→宾语），可有多种类型。
- **使用边界**：关系类型由关系目录定义，不得私自创建未定义关系。关系用于构建知识图谱，支持跨实体查询和导航。
- **易混概念**：与"关联"不同——关联是泛指，关系是精确定义的语义连接；与"引用"不同——引用是单向提及，关系是双向语义连接。
- **项目内示例**：`occurs_at`（地方习俗实例 → 地点）——表示某地方习俗实例发生在某地点。

### 5.4 锚点对象（Anchor Entity）
- **定义**：在知识系统中承担稳定参照作用、要求其他核心记录围绕其建立定位或归属关系的实体。
- **使用边界**：本项目中的首要锚点对象是地点（Place）。所有地方习俗实例必须绑定地点；节日、仪式等抽象词条通过地方习俗实例获得地理落点。
- **易混概念**：与"入口页面"不同——锚点对象是数据结构中的参照实体，不是产品界面的导航入口；与"核心实体"不同——核心实体有多个，锚点对象强调其参照地位。
- **项目内示例**：`Place` 是锚点对象，`CustomInstance` 通过 `occurs_at` 关系绑定到 `Place`。

### 5.5 扩展实体（Extended Entity）
- **定义**：已确定预留前缀和后续职责边界，但在首期不独立展开、不纳入常规发号和模板必做范围的实体。
- **使用边界**：人群角色、器物、信仰观念、时间规则、媒体资料属于扩展实体。首期通常以文本描述、来源子类型或预留前缀引用的方式出现，不作为独立数据对象全面建设。
- **易混概念**：与"首期核心实体"不同——首期核心实体是当前必须稳定落地的 5 个实体；扩展实体是二期才独立建模的对象。
- **项目内示例**：`Media` 在首期作为来源资料的补充或子类型出现，不作为独立实体并行展开。

## 6. 内容状态相关术语

### 6.1 变体（Variant）
- **定义**：同一习俗主题在不同地点、不同时间或不同人群中形成的具体差异形态。变体是地方习俗实例的核心价值所在。
- **使用边界**：变体通过并行的独立地方习俗实例记录体现，不强行统一或判定"正宗"。变体记录应注明差异维度（地点差异、时间差异、人群差异）。
- **易混概念**：与"错误"不同——变体不是错误，是正常的地域多样性；与"版本"不同——版本是时间轴上的修订，变体是空间轴上的差异。
- **项目内示例**：`春节吃饺子`（北方实例）与`春节吃年糕`（南方实例）是同一节日的两个变体。

### 6.2 存续状态（Continuity Status）
- **定义**：描述习俗在特定地点当前传承状况的分类，用于标记习俗的活态程度。
- **使用边界**：存续状态用于地方习俗实例，可选值包括：活跃传承、逐渐式微、已中断、已恢复、仅文献记载等。状态可随时间变化，需记录状态判定时间。
- **易混概念**：与"保护级别"不同——存续状态描述实际传承状况，保护级别是行政管理认定；与"真实性"不同——存续状态不涉及习俗真伪，只涉及是否仍在实践。
- **项目内示例**：`某村舞龙习俗`——标记为"逐渐式微"，注明判定时间 2024 年，说明参与人数减少、年轻传承人缺乏。
