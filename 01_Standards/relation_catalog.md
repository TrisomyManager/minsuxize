## 1. 关系总览

### 1.1 关系设计原则
- **必要性优先**：首期仅定义支撑基本查询和导航的核心关系
- **方向明确**：关系有方向性，主语→宾语，反向关系需单独定义
- **语义清晰**：关系名称准确表达语义，避免歧义
- **可扩展性**：关系命名预留二期扩展空间

### 1.2 首期核心关系（5实体间）
| 关系类别 | 关系数量 | 说明 |
|---------|---------|------|
| 地点内部关系 | 2 | 层级隶属关系 |
| 地点-实例关系 | 2 | 空间锚点关系 |
| 抽象-实例关系 | 4 | 节日/仪式与实例的对应 |
| 实例-来源关系 | 2 | 证据支撑关系 |

### 1.3 二期扩展关系（含扩展实体）
涉及人群角色、器物、信仰观念、时间规则、媒体资料的关系在首期不展开，标记为"二期"。这些关系只有在扩展实体进入独立建模后才进入正式实现范围。

## 4. 变体关系特殊说明

### 4.1 变体关系的设计
地方习俗实例之间的"变体关系"不采用显式关系边，而是通过以下方式表达：
- 同一 Festival 的多个 CustomInstance 自动构成变体集合
- 通过 `variants_notes` 字段描述与其他实例的差异
- 通过对比查询（同一节日、不同地点）发现变体模式

### 4.2 变体关系的查询示例
```
查询：春节在不同地点的变体
方式：找出所有 festival_id = FES-0001 的 CustomInstance
对比：对比它们的 place_id、custom_procedure、participants_description 等字段
```

## 5. 关系与下游模块映射

### 5.1 与图谱展示映射
- 首期核心关系（occurs_at, is_instance_of, realizes, supported_by 等）用于构建基础图谱
- 地点层级关系（has_parent/has_child）用于构建地理层级树
- 抽象-实例关系用于构建“抽象词条→地方习俗实例”的导航路径

### 5.2 与查询需求映射
| 查询需求 | 使用关系 |
|---------|---------|
| 某地点有哪些习俗 | has_instance |
| 某节日在不同地点的差异 | has_instance（多个）+ 字段对比 |
| 某仪式的地域分布 | realized_in |
| 某来源支撑哪些事实 | supports |
| 某实例的来源是什么 | supported_by |
