## 1. 使用目的
本模板用于不同 AI、不同会话、不同编辑者之间的标准化交接，确保后续执行者能在不重复摸底的前提下继续推进当前模块。

适用场景：
- 单轮任务结束后的正式交接。
- 同一模块拆分为多个子轮次时的中间交接。
- 关键节点整理审核前的移交。
- 只读审阅、修复、导出全文等模式切换前的状态说明。

## 2. 填写规则
- 每次交接只对应一个当前模块编号。
- 文件路径必须写相对仓库路径，不得写“若干文件”“相关文档”等模糊表述。
- 已完成内容与未完成内容必须分开写，不得混在一起。
- 风险 / 冲突必须写清是“已确认问题”还是“待确认问题”。
- Git 信息必须至少包含当前分支与建议提交信息。
- 若本轮为只读审阅，必须在“当前工作模式”中明确写出“只读审阅”。
- 若本轮为本地直改，必须在“当前工作模式”中明确写出“本地直改 + 结构化回报”。

## 4. 精简交接模板

## 5. 不合格交接示例
以下写法不合格：
- “都改完了，可以继续做下一个。”
- “相关文件已更新。”
- “大概没问题，后面再看。”
- “字段和关系已经对齐。”

以上写法的问题：
- 没有模块编号。
- 没有文件路径。
- 没有已完成与未完成的区分。
- 没有风险说明。
- 没有 Git 信息。

## 6. 推荐交接时机
- 完成一个模块后立即交接。
- 一轮只完成部分内容时，先交接再退出。
- 切换到只读审阅前，先交接当前修改状态。
- 需要他人接手修复冲突时，先交接冲突范围和影响。
