# 中国民俗细则

“中国民俗细则”是一个面向中国传统民俗记录、整理、审核与公开发布的网站项目。

项目目标不是简单收集“节日介绍”，而是尽可能细致地记录各地，尤其是乡镇、村落层面的真实民俗实践，让后人能够看到：

- 某个地方在某个农历日期具体做什么
- 仪式由谁主持、按什么顺序进行
- 会使用哪些物品、食物、器具、供品
- 有哪些口传内容、禁忌、说法与地方解释
- 这些习俗今天还是否存在，是否发生变化

这个项目参考维基百科式协作模式，但采用更严格的审核机制：用户可以提交信息，经过审核后才对外公开。

## 核心原则

- 以地方性为核心：记录尽量细化到省、市、县、乡镇、村落
- 以时间性为骨架：围绕农历日期、节气、节日、人生礼俗组织内容
- 以证据为基础：每条信息尽量附带口述来源、文献来源、影像来源或实地记录
- 以版本为保障：保留条目修订历史，避免信息被无痕覆盖
- 以尊重为前提：避免用“迷信”“落后”等标签化语言处理民间信仰与风俗

## 网站应承载的内容

- 行政区划与地理层级
- 节日与农历时间体系
- 地方民俗条目
- 仪式流程
- 民俗物件与供品
- 口述资料与文献来源
- 用户投稿与审核记录
- 条目修订历史

## 建议的内容组织方式

不要把“地区”和“节日”混成单一页面，而是拆成几个独立对象：

1. 地区
2. 节日/时间节点
3. 民俗条目
4. 来源证据
5. 投稿与审核

这样同一个节日可以在不同村落下复用，同一个村落也可以挂载多个节日和多种民俗。

## 推荐的 MVP 范围

第一阶段先只做最有价值的闭环：

- 支持省、市、县、乡镇、村五级地区录入
- 支持农历节日与节气录入
- 支持民俗条目创建、编辑、审核、发布
- 支持来源证据上传或登记
- 支持用户投稿
- 支持审核员退回、修改、通过
- 支持公开检索与浏览

先不要急着做太重的功能：

- 不必一开始就做复杂地图系统
- 不必一开始就做移动端 App
- 不必一开始就做过度社交化互动
- 不必一开始就做自动 AI 总结替代人工审核

## 用户角色建议

- 游客：浏览公开内容
- 注册用户：提交新条目、补充材料、发起修订建议
- 审核员：核查来源、修改格式、决定是否发布
- 管理员：维护地区库、节日库、用户权限、敏感内容规则

## 关键页面建议

- 首页：项目介绍、最新收录、重点节日、重点地区
- 地区页：某省/市/县/乡/村的民俗总览
- 节日页：某个节日在全国或某地区的变体
- 条目页：具体民俗内容、来源、修订记录
- 投稿页：结构化提交表单
- 审核后台：待审核列表、差异对比、发布记录

## 运行原型

在项目根目录运行：

```powershell
dotnet run --project .\src\MinsuXize.Web\MinsuXize.Web.csproj
```

如果想先编译检查：

```powershell
dotnet build .\src\MinsuXize.Web\MinsuXize.Web.csproj
```

详细设计见：[docs/site-blueprint.md](D:\DeskTop\minsuxize\docs\site-blueprint.md)

## 数据持久化

当前原型默认使用本地 SQLite 数据库：

- 数据库文件：`src/MinsuXize.Web/App_Data/minsuxize.db`
- 首次启动会自动建库并写入示例地区、节日、条目、来源和投稿数据
- 投稿与审核状态会持久化保存在这个数据库里

后续切换 PostgreSQL 时，主要替换数据库 provider、连接字符串和迁移流程即可，页面与仓库接口可以继续沿用。
